How Pay on Account Details Works?
Pay on account is a feature which gives previlege to the customers to make credit purchases in the website. In order to activate the pay on account feature for a customer, they have to place pay on account request from the customer registration page or from the profile editing page. Administrator can activate or reject the request from the edit page of such customers. While activating the pay on account feature for a customer, the maximum credit limit and also the billing cycle day should be selected. This can be done from the customer edit page. Once this feature is activated for a customer, the pay on account payment option will be listed in various payment sections in the website for that customer.
 
1. List Pay on Account Details
2. Account Summary
3. View Statements
4. View Statement Details
 
List Pay on Account Details
Pay on account details page shows a consolidated view of pay on account details for all customers for which this feature is/was active.
Pay on account details can be searched based on Customer name or Email id, Credit limt, Used Credit Limit and Pay on Account Status. Any combination of these can be used as search criteria. Number of customers to be shown in each of the pages and also the sort order to be used in search result can be specified. Use "Go" button to begin the search operation.
By default the status field will have the value "Any".
In order to view the pay on account summary of a particular customer, click on the "Account Summary" link of that customer in the list.

Account Summary
The account summary of selected customer will be displayed in this page. The page will display the Credit limit and also the Current Account Balance for the customer.
The page also shows the list of unbilled transactions and the next statement date. The unbilled transactions are the transactions which are to be reflected in the next statement.
The key fields are as follows.
1. Date
Date on which the transaction took place.
2. Details
The details of transaction will be displayed in this column. Three types of transactions can appear in this section
a. Transaction related to checkout
The details of checkout transaction will be displayed with the order id. The format will be "Order Id x", where x is the order id. When clicked on this details, user will be taken to the respective orders details page.
b. Transaction related to voucher purchase
The details of voucher purchase transaction will be displayed with the voucher id. The format will be "Voucher Id x", where x is the gift voucher id id. When clicked on this details user will be taken to the respective gift voucher edit page.
c. Payment made by customer for their credit purchases.
When customer make a payment against their credit purchases, it will be listed in this section. It is also possible to record a payment for a particular customer from this page. Such transactions will have the details "Payment - Thank you". Since it is a credit transaction a "Cr." symbol will be displayed next to the the amount of such transactions in the listing section.
To record a payment for current customers against the credit purchase click on the "Make Payment" button. When this button is clicked, a new section will be displayed where the following things will be asked for.
a. Amount
The amount being paid should be set in this section.
b. Payment Method
The method by which the payment is made from the console is to be selected from this dropdown box.
c. Additional Details
If any additional detail is to be saved along with this payment such as a note, then it can be specified in this section.
Click on "Make Payment" button to make the payment. There is no gateway involved when payment is made from the console area. The system just records the payment details in the pay on account section of selected customer.
Click on "Cancel" button if payment is not to be made at this moment.
In order to view previous statements, click on "View Statements" button. When this button is clicked, a new page will be displayed which shows the list of all the statement of current customer with the latest statement at the top.

View Statements
List of statement which have been generated for current customer will be listed in this page. Pay on Account statements will be automatically generated and send to respective customers depending of the billing cycle day set for them. Searching can be done by specifying the from and to dates. The number of statements to be displayed in each of the pages and also the sort order for statements can be set from this section. Click on "Go" button to begin the search operation.
To view the trasactions of a particular statement, click on the "Statement Date" column in the list.
Back to Top
View Statement Details
The list of transaction involved in a selected statement will be listed in this section. The transactions will be listed in the order in which it had occured.
The Statement Date, Credit Limit, Current Account Balance and also the Closing Balance of current statement will be displayed in this page.
