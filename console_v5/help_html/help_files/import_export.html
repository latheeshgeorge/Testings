IMPORT
Data to be imported should be in CSV format. A button “Generate Template” is shown with each of the import options. When clicked on this button, a CSV files will be asked for downloading. This CSV file will have just the header row. This is to make the console user aware of the format and the order in which the data is to be placed in the importing CSV file. Data being imported should be in the same order and data type as that shown in the “Generate Template” CSV file, otherwise system will reject it, or may result is wrong data getting placed in wrong fields. That is; the order of the fields in CSV file is very important.
If “Import” is selected from the drop down box, then another drop down box “Import What?” will be displayed. This drop down box have the following values

1. Categories
If “Categories” is selected then the following options will be displayed
1. Select the Parent Category
This drop down box will show all the categories existing in the site. If the categories being imported is to be placed under a particular category, that parent category should be selected in this drop down box. By default, “­­­Root Level­­” will be selected in the drop down box, which means the categories being imported will be root level categories.
2. Select the Category Group
Category groups are the containers for holding categories in the site. If the categories being imported is to be placed in category groups, those groups should be selected. It is possible to place same category in more than one category group.
3. Select file to import
The CSV that is to be imported has to be selected using the browse button
4. Whether the importing file Have Header?
If the first line in the CSV file that is being imported is a header row, then this should be ticked. If the CSV file have a header row and if this option is not ticked, then the header row will be considered as data and the console will try to import it as a category and may result in error.
Back to Top
2. Products
If “Products” is selected then the following options will be displayed
1. Select the Category
The category to which the products being imported is to be mapped should be selected in this drop down box.
2. Select file to import
The CSV file with the product details should be selected using the browse button.
3. Whether the importing file Have Header?
If the first line in the CSV file that is being imported is a header row, then this should be ticked. If the CSV file have a header row and if this option is not ticked, then the header row will be considered as data and the console will try to import it as a product and may result in error.
Back to Top
3. Shop by brand
If “Shop by Brand” is selected, then the following options will be displayed
1. Select the Parent Shop
If the shops being imported is to be placed under any of the shops already existing in the site, then that shop should be selected in this drop down box.
2. Select the Shop Group
Shop groups are the containers which holds shops in the site. If the shops being imported are to be placed in shop by brand groups, those groups should be selected in this list box. It is possible to map same shop to more than one shop groups.
3. Select file to import
The CSV file with the product details should be selected using the browse button
4. Whether the importing file Have Header?
If the first line in the CSV file that is being imported is a header row, then this should be ticked. If the CSV file have a header row and if this option is not ticked, then the header row will be considered as data and the console will try to import it as a shop and may result in error .

4. Customers
If “Customers” option is selected, then the options available are “Select file to import” and “Whether the importing file Have Header?”, which allows to select the CSV to be imported and also the option to specify whether the first row in the CSV is a header row or not.
EXPORT
Export can be done in following formats
1. CSV
2. HTML
3. SQL Dump
The drop down box “ Export What?” will have the following values
Back to Top
1. Categories
When “Categories” is selected form the drop down box, the following options will be displayed
1. Select the Parent Category
If parent categories are selected, then the categories under the selected parent categories will only be exported.
2. Select the Category Group
If a category group is selected, then categories that are mapped to the selected category groups only will be exported.
If both “Parent Category” and “Category Group” are selected, then both the conditions will be considered to decide the categories to be exported.
3. Select the Fields to be exported
The names of fields that are to be included in the export file should be selected from this list box. Use “Ctrl” button to make multiple selection.
4. All Fields Required
If all fields are to be exported, then tick mark this field.
5. Sort By
The sort order in which the categories being exported should be done can be selected from this drop down box.
6. Output Format
User can decide the format in which the export shou ld be done. The default format will be CSV.
