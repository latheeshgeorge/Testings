List Customers
Customers can be searched based on Customer Name, Customer Email, Member of a specific business, Mapped to a specific department and Pay on account status. Any combination of these can be used in the search criteria. Number of customers to be shown in each of the pages and also the sort order to be used in search result can be specified. Use "Go" button to begin the search operation.
Use "Add" link to go the customer add page. There are 2 ways to go to the edit page of an existing customer. One method is by clicking the customer name and another by tick marking the required customer and pressing the "Edit" icon.
Delete operation can be done to more than one customers at a time. Tick mark the customers to be deleted and press the "Delete" icon.
Hidden Status of customers can be changed directly from the listing page. Tick mark the customers which need status changing. Select the required status from the drop down box provided at the section "Change Status" and press the "Change" button.
If the import / export feature is active for your site, then option to export customers will be displayed under the list of customers. There are 2 types of customer export
1. Export Selected Customers
If this option is selected, the tick marked customers will be exported.
2. Export All Customers
If this option is selected, all customers will be exported.
When clicked on the "Export Now" button, you will be taken to the main import / export section and there you will be able to select the following things related to customers being exported
1. Fields to be included in the exported file.
2. How the customers should be sorted in the exported file
3. Output Format of export file. The available format as CSV, HTML and SQL Dump.

Add Customers
Customers can be added through this section.
Key Fields
Personal Details
Customer title, First name, postcode and phone are compulsory field in this block.
Country is a dropdown box. The values listed in this drop down box can be managed from the "Country" link in General Settings section (Settings » General Settings » Country). When a country is selected in this page, the list of states which are under the selected country will be displayed in a new dropdown box. If the required state is not there in the list provided in the drop down box, then choose the "- Other -" option. This will show a new textbox where the name of the state can be specified. When Customer details is saved, the new state will get automatically placed under the selected country and from next time onwards it will get automatically get displayed in the state drop down box.
Related Topic: List of countries and states to be used in the site can be managed directly from the general settings area. Settings » General Settings » Country can be used to manage country. Settings » General Settings » State can be used to manage the state list.
Activate
This field decides whether the current customer account is active or not. When a customer get registered from the site, this will be ticked by default.
Customer Login
Email Id
Email id will be the user name for the customer login.
Password and confirm passwords should be identical.
Affiliate
This section allows to set the details related to affiiate customers.
Affiliate
If the current customer is a affiliate or request to be an affiliate then this will be ticked.
Approved Affiliate
If the affiliate request from customer is approved, then this will be ticked.
Affiliate Commission
The commision % for this affiliated customer can be set in this text box.
Other
Other settings for current customer can be set in this section.
Bonus Point
The bonus points for current customer can be set directly in this textbox. When customer purchase a product which have bonus points, those bonus points will get added to this textbox of that customer.
Discount
If any direct discount is to be given for current customer, then the discount % can be set in this textbox. The discount set in this textbox will be ignored if customer is part of any customer discount group.
Shop / Branch
If current customer is related to a specific branch or shop, then that can be set from this section. By default this will have the value "Web". When a customer get registered in the site, the branch "Web" will be set to them. Branches for the current website can be managed form Settings » Branches section.
Allow Product Discount
This field decides whether any discount is applicable to current customer. If this is ticked, the discount (product discount, groups discount etc) will be applicable for current customer. If not ticked, no discount will be applicable to current customer.
Use Bonus Points
Customer will be allowed to spend their bonus points only if this is ticked. The option to spend the bonus points will be displayed for customer in cart page only if this is ticked.
To receive newsletters please select the following
If customer would like to recieve newsletters, then they will have to tick mark the news letter groups listed under this section. This section will be visible only if newsletter groups exists added to the site from console area. Newsletter groups can be manged for the site from the location Settings » Manage Newsletters » Newsletter Group.

Pay on Account
The pay on account details for current customer can be managed from this section. Current Status field decides the current status of pay on account for the customer being edited.
Key Fields
Credit Limit
The credit limit for current customer can be set in this textbox. The value set in this textbox is the maximum value of credit purchases allowed to current customer.
Credit Balance
This field indicates the credits remaining for current customer. For example if the credit limit for the customer is £5,000 and they have used the pay on account for purchases for £3,000, then the credit balance field will display £2000 (i.e £5000-£3000).
Credits Used
Each time when the customer make a purchase using their pay on account, that amount will get added to the credits used field. This is the total outstanding amount for the current customer which is to be paid to the site owner at a later date.
Bill Date
The billing cycle date for current customer can be set using this drop down box. The day of month should be selected. By default 1 will be value in the drop down box. Pay on account statement will be generated on the selected day of evey month and will be send to the respective customers automatically.
Last Statement Date
The date of last statement send to the customer will be displayed in this section (if any)
Current Status
Pay on account status of current customer can be obtained from the value selected in this drop down box. The values shown in the drop down box and its meaning are as follows
Not Requested :- Pay on account feature is not yet requested by the customer. This will be the default value for all the customers.
Requested :- When customer place a pay on account request, but no action has been taken by console administrator yet, then the current status field will show this status.
Rejected :- If the pay on account request send by customer is rejected, then the current status field will show the Rejected value. When this option is selected from the drop down box, a text area will be displayed to enter the reason for rejection.
Active :- If pay on account request is accepted and is actived, then the current status drop down box will show this status.
Inactive :- If the pay on account for certain customer have to be made inactive due to certain reason, then this option can be selected.
When customers place pay on account request, the value of current status field will be displayed as "Requested" for such customers in their edit page. Also in the console home page, a notification will be displayed to incidate that "n" number of customers have been requested for pay on account. This will be a link in the console home page which will take the console user to the page where the list of all customers who have made the pay on account request will be displayed. If the pay on account request can be accepted, then choose the "Active" status from the current status drop down box. Choose the "Rejected" option in case if pay on account request is to be rejected. the details will be saved when clicked on the "Update" button.

Tab: Favourite Categories
While viewing the category details page in the website, registered users have options to add the category being viewed to their favourite category list. Categories added as such will be listed in this section.
Use "Assign More " button to assign more categories to the favourite category list of current customer. When clicked on "Assign More " button a page which shows the list of categories will be displayed. Select the required categories and click the "Assign Selected" button to make the assignment.
"Un Assign" button can be used to unassign any of the categories already assigned to the favourite category list of current customer. Tick mark the categories to be unasssigned and click on the "Un Assign" button.

Tab: Favourite Products
While viewing the product details page in the website, registered users have options to add the product being viewed to their favourite products list. Products added as such will be listed in this section.
Use "Assign More " button to assign more products to the favourite product list of current customer. When clicked on "Assign More " button a page which shows the list of products will be displayed. Select the required products and click the "Assign Products" button to make the assignment.
"Un Assign" button can be used to unassign any of the products already assigned to the favourite product list of current customer. Tick mark the products to be unassigned and click on the "Un Assign" button.
